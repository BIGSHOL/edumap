## 1. 시스템 아키텍처

### 1.1 고수준 아키텍처

```
┌──────────────────────────────────────────────────────────────────┐
│                        Vercel (배포)                              │
│  ┌──────────────────────────────────────────────────────────┐    │
│  │              Next.js App (FE + BE 통합)                   │    │
│  │  ┌─────────────┐     ┌──────────────┐                    │    │
│  │  │   React UI   │────▶│  API Routes  │                    │    │
│  │  │  (Pages)     │     │  (Server)    │                    │    │
│  │  └─────────────┘     └──────┬───────┘                    │    │
│  └──────────────────────────────┼────────────────────────────┘    │
│                                 │                                  │
└─────────────────────────────────┼──────────────────────────────────┘
                                  │
                    ┌─────────────┼─────────────┐
                    │             │             │
              ┌─────▼────┐ ┌─────▼────┐ ┌─────▼────┐
              │ Supabase │ │ Claude   │ │ 공공데이터│
              │ (DB)     │ │ API      │ │ API      │
              └──────────┘ └──────────┘ └──────────┘
                                         ├─ 학교알리미
                                         ├─ 나이스
                                         └─ 지도 API
```

### 1.2 컴포넌트 설명

| 컴포넌트 | 역할 | 왜 이 선택? |
|----------|------|-------------|
| Next.js (FE) | 대시보드 UI, 지도/차트 시각화, 리포트 뷰 | React 기반 SSR/SSG 지원, 대회 데모 시 빠른 초기 로딩 |
| Next.js API Routes (BE) | 공공 API 프록시, Claude API 호출, 데이터 처리 | FE/BE 통합으로 배포 단순화, Vercel 원클릭 배포 |
| Supabase (DB) | 수집된 공공데이터 저장, 분석 결과 캐싱 | 무료 티어로 대회 데모 충분, PostgreSQL 기반, 웹 대시보드 |
| Claude API | 대상별 자연어 리포트 생성 | 한국어 품질 최우선, 교육 도메인 이해도 높음 |
| 공공데이터 API | 학교/교육 원본 데이터 수집 | 대회 필수 요구사항 |
| 지도 API | 지역별 데이터 시각화 | GapMap 모듈 시각화, 심사 임팩트 |

## 2. 권장 기술 스택

### 2.1 프론트엔드

| 항목 | 선택 | 이유 | 벤더 락인 리스크 |
|------|------|------|-----------------|
| 프레임워크 | Next.js 14+ (App Router) | SSR/SSG 지원, Vercel 최적화, React 생태계 | 중간 (Vercel 의존) |
| 언어 | TypeScript | 공공 API 응답 타입 안전성 필수, 컴파일 타임 에러 방지 | 없음 |
| 스타일링 | Tailwind CSS | 빠른 UI 개발, 정부 스타일 커스터마이징 용이 | 낮음 |
| 차트 | Recharts | React 네이티브, 교육 데이터 시각화에 적합 | 낮음 |
| 지도 | Leaflet + react-leaflet | 오픈소스 무료, 한국 지도 타일 지원 | 낮음 |
| HTTP 클라이언트 | fetch (내장) | Next.js 캐싱과 통합, 추가 의존성 불필요 | 없음 |

### 2.2 백엔드 (Next.js API Routes)

| 항목 | 선택 | 이유 | 벤더 락인 리스크 |
|------|------|------|-----------------|
| 런타임 | Next.js API Routes (App Router) | FE와 통합, Vercel 서버리스 함수로 자동 배포 | 중간 |
| 언어 | TypeScript | FE와 타입 공유, 공공 API 응답 타입 통일 | 없음 |
| ORM | Prisma | Supabase PostgreSQL 연동, 타입 안전 쿼리, 마이그레이션 관리 | 낮음 |
| 검증 | Zod | TypeScript 네이티브 스키마 검증, API 응답 파싱 | 낮음 |
| AI SDK | @anthropic-ai/sdk | Claude API 공식 SDK, 스트리밍 지원 | 중간 |

### 2.3 데이터베이스

| 항목 | 선택 | 이유 |
|------|------|------|
| 메인 DB | PostgreSQL (Supabase) | 무료 티어 500MB, 웹 대시보드, REST API 제공 |
| 파일 저장 | 로컬 CSV/JSON → DB 적재 | 공공데이터 파일 다운로드 후 파싱하여 DB에 적재 |
| 캐시 | Supabase (테이블) | 별도 Redis 불필요, API 응답 캐싱용 테이블로 충분 |

### 2.4 인프라

| 항목 | 선택 | 이유 |
|------|------|------|
| 호스팅 | Vercel | Next.js 최적화, 무료 티어, 원클릭 배포, 커스텀 도메인 |
| 환경변수 | Vercel Environment Variables | 배포 환경별 관리, .env.local로 로컬 개발 |
| 모니터링 | Vercel Analytics (무료) | 기본 성능 모니터링, 대회 데모에 충분 |

## 3. 비기능 요구사항

### 3.1 성능

| 항목 | 요구사항 | 측정 방법 |
|------|----------|----------|
| 페이지 로딩 | < 3s (FCP) | Lighthouse |
| API 응답 (데이터 조회) | < 1s (P95) | Vercel Analytics |
| AI 리포트 생성 | < 30s (스트리밍) | 수동 측정 |
| 지도 렌더링 | < 2s (초기 로딩) | 수동 측정 |

### 3.2 보안

| 항목 | 요구사항 |
|------|----------|
| API 키 보호 | 모든 외부 API 키는 서버 사이드(API Routes)에서만 사용 |
| HTTPS | 필수 (Vercel 자동 제공) |
| 입력 검증 | Zod로 모든 요청 파라미터 서버 측 검증 |
| 개인정보 | 학생 개인정보 절대 노출 금지 — 학교 단위 집계만 |
| CORS | Next.js 기본 설정 (같은 도메인이므로 별도 설정 불필요) |

### 3.3 확장성

| 항목 | 현재 (MVP) | 목표 (v2) |
|------|------------|-----------|
| 동시 사용자 | 데모: 10명 | 100명 |
| 데이터 용량 | 100MB (샘플) | 1GB (전국) |
| API 호출 | 학교알리미 + 나이스 | + KESS + EDSS |

## 6. 데이터 생명주기

### 6.1 원칙

- **최소 수집**: 학교 단위 집계 데이터만 수집 (개인정보 없음)
- **출처 표기**: 모든 데이터에 출처 표기 필수 (학교알리미, KESS 등)
- **캐싱**: 공공 API 응답은 DB에 캐싱하여 재호출 방지

### 6.2 데이터 흐름

```
공공데이터 API/파일 → 수집 스크립트 → 파싱/정제 → Supabase DB 적재
                                                          │
                                                    ┌─────┴─────┐
                                                    │ API Routes │
                                                    │ (조회/분석)│
                                                    └─────┬─────┘
                                                          │
                                                    ┌─────▼─────┐
                                                    │ Claude API │
                                                    │ (리포트)   │
                                                    └─────┬─────┘
                                                          │
                                                    ┌─────▼─────┐
                                                    │  React UI  │
                                                    │ (리포트 뷰)│
                                                    └───────────┘
```

| 데이터 유형 | 보존 기간 | 삭제/갱신 |
|------------|----------|----------|
| 공공데이터 원본 (DB) | 영구 | 연 1회 갱신 |
| 생성된 리포트 | 영구 | 삭제 없음 (캐싱) |
| API 응답 캐시 | 7일 | 자동 만료 |

## 7. 테스트 전략 (Contract-First TDD)

### 7.1 개발 방식

Next.js 통합 프로젝트이므로, BE/FE 분리 대신 **기능 단위 TDD**를 적용합니다.

```
1. API 계약 정의 (타입 + Zod 스키마)
2. API Route 테스트 작성 (🔴 RED)
3. 컴포넌트 테스트 작성 (🔴 RED)
4. API Route 구현 (🟢 GREEN)
5. 컴포넌트 구현 (🟢 GREEN)
6. E2E 테스트로 전체 플로우 검증
```

### 7.2 테스트 피라미드

| 레벨 | 도구 | 커버리지 목표 | 위치 |
|------|------|-------------|------|
| Unit | Vitest | ≥ 80% | src/**/__tests__/ |
| Integration | Vitest + MSW | Critical paths | src/**/__tests__/ |
| E2E | Playwright | Key user flows | e2e/ |

### 7.3 테스트 도구

| 도구 | 용도 |
|------|------|
| Vitest | 단위/통합 테스트 실행 (Next.js 호환) |
| React Testing Library | 컴포넌트 테스트 |
| MSW (Mock Service Worker) | 공공 API / Claude API 모킹 |
| Playwright | E2E 브라우저 테스트 |

## 8. API 설계 원칙

### 8.1 Next.js API Routes 규칙

| 메서드 | 용도 | 예시 |
|--------|------|------|
| GET | 데이터 조회 | GET /api/schools?region=서울 |
| POST | 리포트 생성 요청 | POST /api/report |

### 8.2 응답 형식

**성공 응답:**
```json
{
  "data": { ... },
  "meta": {
    "source": "학교알리미",
    "updatedAt": "2026-03-01"
  }
}
```

**에러 응답:**
```json
{
  "error": {
    "code": "API_UNAVAILABLE",
    "message": "학교알리미 API 응답이 지연되고 있습니다. 캐시된 데이터를 사용합니다.",
    "fallback": true
  }
}
```

### 8.3 공공 API Fallback 전략

```
1차: 공공 API 실시간 호출
  ↓ (실패 시)
2차: DB 캐시된 데이터 반환 (마지막 성공 응답)
  ↓ (캐시 없음)
```
