# User Flow (사용자 흐름도) — 에듀맵 (EduMap)

> Mermaid 플로우차트로 핵심 기능의 주요 여정을 표현합니다.

---

## MVP 캡슐

| # | 항목 | 내용 |
|---|------|------|
| 1 | 목표 | 교육 공공데이터를 AI로 분석하여 학습격차를 조기 탐지하고, 누구나 이해할 수 있는 자연어 리포트를 자동 생성하여 대회 수상 |
| 2 | 페르소나 | 교육청 정책담당자, 학교 교사, 학부모 |
| 3 | 핵심 기능 | FEAT-1: InsightReport (AI 자연어 리포트 생성) |
| 4 | 성공 지표 (노스스타) | 대회 심사위원 평가 — 수상 |
| 5 | 입력 지표 | AI 리포트 품질 점수, 데이터 통합 정확도 |
| 6 | 비기능 요구 | 공공 API 응답 실패 시 fallback 처리 |
| 7 | Out-of-scope | 수익화, 모바일 앱, 개인 학생 데이터, 사용자 인증 |
| 8 | Top 리스크 | 공공데이터 API 불안정 |
| 9 | 완화/실험 | 로컬 캐시 + 샘플 데이터로 데모 대비 |
| 10 | 다음 단계 | 학교알리미 API 연동 및 데이터 수집 |

## 1. 전체 사용자 여정 (Overview)

```mermaid
graph TD
    A[웹 접속] --> B[FEAT-0: 메인 대시보드]
    B --> C{무엇을 할까?}

    C -->|학교/지역 검색| D[학교/지역 선택]
    C -->|지도에서 탐색| E[지도 뷰에서 클릭]
    C -->|대시보드 개요 확인| F[전국 현황 차트 확인]

    D --> G[학교/지역 상세 정보]
    E --> G
    F --> G

    G --> H{리포트 생성?}
    H -->|Yes| I[FEAT-1: 리포트 유형 선택]
    H -->|No| B

    I --> J[AI 리포트 생성 중...]
    J --> K{생성 성공?}
    K -->|Yes| L[리포트 확인]
    K -->|No| M[에러 안내 + 재시도]
    M --> I

    L --> N{다른 리포트?}
    N -->|Yes| I
    N -->|다른 학교| B
    N -->|No| O[종료]
```

## 2. FEAT-0: 메인 대시보드 플로우

```mermaid
graph TD
    A[웹 접속] --> B[메인 대시보드 로딩]
    B --> C{데이터 로딩 성공?}

    C -->|Yes| D[전국 개요 표시]
    C -->|No - API 실패| E[캐시 데이터 표시 + 안내 배너]
    C -->|No - 캐시도 없음| F[샘플 데이터 표시 + '샘플' 표시]

    D --> G[전국 지도 뷰]
    E --> G
    F --> G

    G --> H[지역별 현황 차트]
    H --> I[학교 검색 바]

    I --> J{검색 입력}
    J -->|학교명| K[학교 검색 결과]
    J -->|지역명| L[지역 검색 결과]

    K --> M[학교 상세 페이지]
    L --> N[지역 상세 페이지]
```

## 3. FEAT-1: InsightReport — AI 리포트 생성 플로우

```mermaid
graph TD
    A[학교/지역 상세 페이지] --> B[리포트 생성 버튼 클릭]
    B --> C[리포트 유형 선택]

    C --> D{대상 선택}
    D -->|정책담당자용| E["policy 리포트 요청"]
    D -->|교사용| F["teacher 리포트 요청"]
    D -->|학부모용| G["parent 리포트 요청"]

    E --> H[관련 데이터 수집]
    F --> H
    G --> H

    H --> I{데이터 수집 성공?}
    I -->|Yes| J[Claude API 호출 - 리포트 생성]
    I -->|Partial| K[부분 데이터로 진행 + 안내]
    I -->|No| L[fallback: 캐시/샘플 데이터 사용]

    K --> J
    L --> J

    J --> M[리포트 스트리밍 표시]
    M --> N[리포트 완성]

    N --> O{추가 액션}
    O -->|다른 유형 리포트| C
    O -->|다른 학교/지역| P[메인 대시보드로]
    O -->|리포트 저장/공유| Q[PDF 다운로드 / 링크 복사]
```

## 4. FEAT-2: EarlyAlert — 조기경보 플로우 (Phase 2)

```mermaid
graph TD
    A[메인 대시보드] --> B[EarlyAlert 탭 클릭]
    B --> C[전국 위험도 히트맵 표시]

    C --> D{지역/학교 선택}
    D -->|지도 클릭| E[해당 지역 위험도 상세]
    D -->|검색| F[학교 위험도 상세]

    E --> G[위험도 스코어 0~100 표시]
    F --> G

    G --> H[주요 기여 요인 랭킹]
    H --> I{리포트 연계?}
    I -->|Yes| J[FEAT-1: 리포트 생성으로 이동]
    I -->|No| K[다른 학교/지역 탐색]
    K --> C
```

## 5. 에러 처리 플로우

```mermaid
graph TD
    A[에러 발생] --> B{에러 유형?}

    B -->|공공 API 실패| C[캐시 데이터 사용 + 안내 배너]
    B -->|Claude API 실패| D[재시도 안내 + 타임아웃 연장]
    B -->|네트워크 오류| E[연결 재시도 안내]
    B -->|데이터 없음| F[해당 학교/지역 데이터 미제공 안내]

    C --> G[서비스 정상 이용 가능]
    D --> H{재시도?}
    H -->|Yes| I[Claude API 재호출]
    H -->|No| J[나중에 다시 시도 안내]

    E --> K{재시도?}
    K -->|Yes| L[페이지 새로고침]
    K -->|No| J

    F --> M[다른 학교/지역 탐색 유도]
```
