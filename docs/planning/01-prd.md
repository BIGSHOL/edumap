## 1. 문제 정의

### 1.1 현재 상황 (As-Is)

교육 데이터는 마치 **여러 서랍에 흩어져 있는 퍼즐 조각**과 같습니다. 학교알리미, 나이스, 교육통계서비스, 에듀데이터서비스 등 여러 기관에 데이터가 분산되어 있어, 한 학교나 지역의 교육 현황을 종합적으로 파악하려면 일일이 찾아다녀야 합니다.

설령 데이터를 모아도 **숫자와 표의 나열**일 뿐, "왜 이 지역에서 격차가 생기는지", "어떤 조치가 필요한지"를 알려주는 사람은 없습니다. 전문가가 아니면 데이터를 읽는 것조차 어렵습니다.

### 1.2 목표 상황 (To-Be)

에듀맵은 **흩어진 퍼즐 조각을 하나의 그림으로 완성**해주는 서비스입니다. AI가 여러 공공데이터를 자동으로 연결하고 분석하여, 교사에게는 "우리 학교 상황", 학부모에게는 "우리 아이 학교 환경", 정책담당자에게는 "지역 교육 격차 현황"을 **이야기처럼 풀어주는 리포트**를 자동으로 생성합니다.

### 1.3 왜 지금인가? (Why Now)

- **제8회 교육 공공데이터 AI 활용대회** 출품 (2026년 5월 제출 목표)
- 교육 격차 문제에 대한 사회적 관심 증가
- Claude AI의 한국어 자연어 생성 품질이 실용 수준에 도달

## 2. 사용자 페르소나

### 2.1 주요 페르소나 A — 정책담당자

| 항목 | 내용 |
|------|------|
| 이름/별칭 | "데이터를 원하는 김과장" |
| 나이/직업 | 40대, 시도교육청 교육정책과 담당 |
| 핵심 니즈 | 지역별 교육 격차를 데이터로 파악하고 예산 배분 근거 확보 |
| 불편 포인트 | 여러 사이트를 돌아다니며 수작업으로 데이터 취합, 보고서 작성에 시간 소모 |
| 사용 상황 | 연간/분기 정책 수립 시, 예산 편성 근거 자료 필요 시 |

### 2.2 주요 페르소나 B — 교사

| 항목 | 내용 |
|------|------|
| 이름/별칭 | "학생이 걱정되는 이선생" |
| 나이/직업 | 30대, 중학교 담임교사 |
| 핵심 니즈 | 우리 학교 학생들의 학습 격차 현황과 원인 파악 |
| 불편 포인트 | 체감적으로는 격차가 느껴지지만 데이터로 확인할 방법이 없음 |
| 사용 상황 | 학기 초 학생 현황 파악, 학부모 상담 자료 준비 |

### 2.3 주요 페르소나 C — 학부모

| 항목 | 내용 |
|------|------|
| 이름/별칭 | "우리 아이 학교가 궁금한 박엄마" |
| 나이/직업 | 40대, 초등학생 자녀를 둔 학부모 |
| 핵심 니즈 | 자녀 학교의 학습 환경과 자원 현황을 쉽게 이해 |
| 불편 포인트 | 학교알리미를 봐도 숫자만 나열되어 무슨 의미인지 모름 |
| 사용 상황 | 학교 선택 시, 학부모 상담 전 학교 현황 확인 |

### 2.4 페르소나의 하루 (시나리오)

**김과장의 오후**: 내년도 교육 예산 편성 회의를 앞두고 지역별 학습격차 현황 자료가 필요합니다. 예전에는 학교알리미, KESS, 통계청 데이터를 각각 다운로드하고 엑셀로 합쳐 분석해야 했습니다. 오늘은 에듀맵에 접속해 "○○시 교육격차 현황"을 검색하고, AI가 생성한 정책 리포트를 다운로드합니다. "교원 1인당 학생 수가 전국 평균 대비 1.3배 높고, 방과후 프로그램 수는 0.6배 수준입니다. 이로 인해 학습격차 위험도가 72점으로 높은 편입니다." — 이 리포트 하나로 회의 자료가 완성됩니다.

## 3. 사용자 스토리

### FEAT-0: 공통 흐름 (접속/메인 대시보드)
- [ ] 사용자로서, 별도 가입 없이 바로 서비스를 이용할 수 있어야 한다. 왜냐하면 대회 데모에서 빠른 접근이 중요하기 때문이다.
- [ ] 사용자로서, 메인 화면에서 전국 학교/지역 개요를 한눈에 볼 수 있어야 한다. 왜냐하면 첫인상이 서비스의 가치를 전달하기 때문이다.

### FEAT-1: InsightReport — AI 자연어 리포트 생성 (MVP)
- [ ] 정책담당자로서, 특정 지역의 교육 격차 현황을 자연어 리포트로 받아볼 수 있어야 한다. 왜냐하면 보고서 작성 시간을 절약하고 데이터 기반 의사결정을 하기 위해서다.
- [ ] 교사로서, 우리 학교의 학습 환경을 분석한 리포트를 받아볼 수 있어야 한다. 왜냐하면 학생 격차의 원인을 이해하고 대응하기 위해서다.
- [ ] 학부모로서, 쉬운 언어로 된 학교 학습환경 인사이트를 받아볼 수 있어야 한다. 왜냐하면 전문 용어 없이도 학교 현황을 이해하고 싶기 때문이다.
- [ ] 사용자로서, 리포트가 "왜 이런 격차가 발생했는가"를 원인 중심으로 설명해야 한다. 왜냐하면 숫자 나열이 아닌 인사이트가 필요하기 때문이다.

### FEAT-2: EarlyAlert — 학습격차 조기경보 (Phase 2)
- [ ] 정책담당자로서, 학교/지역별 위험도 스코어(0~100)를 확인할 수 있어야 한다. 왜냐하면 우선 개입 대상을 선정하기 위해서다.
- [ ] 사용자로서, 위험도에 기여하는 주요 요인 랭킹을 볼 수 있어야 한다. 왜냐하면 어떤 요인이 격차를 만드는지 알아야 대응할 수 있기 때문이다.

### FEAT-3: GapMap — 학습자원 공백 매핑 (Phase 3)
- [ ] 정책담당자로서, 지역별 학습자원 공백을 지도에서 시각화할 수 있어야 한다. 왜냐하면 어디에 자원이 부족한지 직관적으로 파악하기 위해서다.
- [ ] 사용자로서, 공백 지역에 대한 공공 학습자원 연계 추천을 받을 수 있어야 한다. 왜냐하면 문제 발견에서 해결책까지 연결되어야 하기 때문이다.

## 5. 범위 관리

### 5.1 MVP (반드시 포함)
- FEAT-0: 메인 대시보드 (전국 개요)
- FEAT-1: InsightReport — AI 자연어 리포트 생성 (대상별: policy/teacher/parent)
- 학교알리미 API 연동 및 데이터 수집
- 기본 지도/차트 시각화

### 5.2 옵션 (시간 허용 시)
- FEAT-2: EarlyAlert — 위험도 스코어링
- KESS 데이터 연동
- 대시보드 필터링/검색

### 5.3 Non-goals (명시적 제외)
- 수익화 / 결제 시스템 — 대회 출품이 목적
- 모바일 앱 — 웹 전용으로 충분
- 개인 학생 데이터 — 학교 단위 집계 데이터만 사용 (개인정보 보호)
- 실시간 데이터 동기화 — 정적 데이터 분석으로 충분
- 사용자 계정/인증 시스템 — 대회 데모에서 불필요

## 6. 가정과 리스크

### 6.1 핵심 가정

| # | 가정 | 검증 방법 |
|---|------|----------|
| 1 | 학교알리미 API가 안정적으로 데이터를 제공한다 | API 호출 테스트, 응답률 측정 |
| 2 | Claude API가 한국어 교육 도메인 리포트를 충분한 품질로 생성한다 | 프롬프트 실험, 리포트 품질 평가 |
| 3 | 학교 코드로 여러 데이터셋을 정확히 조인할 수 있다 | 샘플 데이터로 매칭률 테스트 |
| 4 | 숫자 나열이 아닌 스토리텔링 리포트가 심사위원에게 차별화된다 | 유사 서비스 분석, 프로토타입 피드백 |

### 6.2 Top 5 리스크 보드

| # | 가정 | 리스크 | 완화 | 트리거 | 대응 |
|---|------|--------|------|--------|------|
| 1 | API 안정성 | 공공 API가 느리거나 다운될 수 있음 | 로컬 캐시 + 샘플 데이터 확보 | API 응답 실패율 > 10% | 캐시된 데이터로 데모 진행 |
| 2 | Claude 품질 | 한국어 교육 리포트 품질이 부족할 수 있음 | 프롬프트 반복 최적화 | 리포트 품질 점수 < 3점 | 프롬프트 재설계 + 후처리 |
| 3 | EDSS 심사 | 학업성취도 데이터 제공 심사가 늦어질 수 있음 | 사전 신청, 대체 데이터 확보 | 제출 2주 전까지 미승인 | EDSS 없이 MVP 완성 |
| 4 | 데이터 조인 | 학교 코드 불일치로 매칭 실패 | 학교 코드 매핑 테이블 사전 구축 | 매칭률 < 90% | 수동 매핑 보완 |
| 5 | 대회 일정 | 개발 일정이 대회 제출일에 맞지 않음 | MVP 범위 최소화, 우선순위 집중 | 제출 1주 전 FEAT-1 미완 | GapMap 제외, 리포트에 집중 |

## 7. 해결 가설과 실험

### 7.1 가설
> 만약 여러 공공데이터를 연결하여 AI가 대상별 맞춤 자연어 리포트를 자동 생성하면, 교사·학부모·정책담당자가 교육 격차를 쉽게 이해하고 데이터 기반 의사결정을 할 수 있을 것이다. 왜냐하면 현재 데이터가 흩어져 있고 숫자만 나열되어 해석이 어렵기 때문이다.

### 7.2 실험 루프

| 단계 | 내용 |
|------|------|
| 실험 | 학교알리미 데이터 + Claude API로 3가지 유형(policy/teacher/parent) 리포트 생성 |
| 관측 | 리포트가 실제로 유용한 인사이트를 담고 있는지, 대상별 언어 수준이 적절한지 |
| 학습 | 리포트 품질 4점 이상이면 성공, 미달 시 프롬프트 재설계 |
| 다음 | EarlyAlert 위험도 스코어를 리포트에 통합 |
